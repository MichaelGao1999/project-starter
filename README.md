# PROJECT STARTER

> 零编程背景也可以享受 Vibe Coding，但需要谨慎不要一路绿灯被 AI 带着跑。从这一份框架开始，每一步你说了算。

---

## 这并非市面上常见的 Harness 工程

市面上很多工具承诺"告诉 AI 目标，剩下的交给它"——但代码由 AI 完成，你又看不懂，心里没底。

**这个框架不一样。** 它提供的是你和 AI 之间的协作协议：

- **阶段口令** — 你决定什么时候进入下一阶段，AI 不会自作主张。@AGENTS.md（§4 指令索引·阶段口令）
- **前置检查** — AI 先告诉你"我发现了什么、我打算怎么做、为什么"，等你确认再动手。@AGENTS.md（§3 RULE-03·哨兵门禁）
- **透明汇报** — 每件事做完，AI 告诉你它做了什么、为什么这样改、有什么风险。@docs/workflows/archive.md（步 7 汇报）
- **存档 / 恢复** — 结束会话前，AI 记录所有决策和进度，下次回来无缝衔接。@docs/workflows/archive.md | @docs/workflows/resume.md

你的判断权始终在你手上。AI 执行，你掌舵。

## 项目特色

五阶段工作流-prompt驱动的五阶段项目搭建流程，能让你立刻从零开始。@docs/workflows/agent-coding-workflow.md

| 阶段 | 你做什么 | AI 做什么 |
|------|---------|----------|
| **一** | 说"阶段一"，描述问题 | 输出需求文档，你确认 |
| **二** | 说"阶段二" | 输出设计文档、反模式检查、选择提醒 |
| **三** | 说"阶段三" | 拆解任务、预估工作量 |
| **四** | 说"阶段四" | 按任务列表逐个编码 |
| **五** | 说"阶段五" | 测试、审查、归档 |

> 这不是机械的步骤，而是你可以随时跳回、跳过或重来的框架。

## 快速开始

如果你有一个项目（新的或已有的），从这里开始：

```bash
# 1. 拿到框架
# 将 project-starter 的全部文件复制到你的项目目录

# 2. 告诉框架你的项目叫什么
# 修改 AGENTS.md → 「1. 项目定位」

# 3. 告诉 AI：恢复
# AI 会读取当前状态，告诉你可以从哪里开始
```

如果你已经配好了 AI 开发助手（Claude Code / 其他），直接对 AI 说 `恢复`。

| `session-log.md` | 每次会话的决策记录，恢复用 |

| `config/github-sync.json` | 如果你有多个项目，这个让经验互通 |
| ... | 更多文件随阶段推进陆续创建 |
